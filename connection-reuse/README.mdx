---
title: Connection reuse
---

The container persists between requests, so module-level state is initialized once and
reused. Module state is per-container and **lost on scale-down or redeploy** — treat it
as a cache, not a store. For durable data use
[KV](https://developers.telnyx.com/docs/edge-compute/kv); for cross-request consistency use
[Stateful Actors](https://developers.telnyx.com/docs/edge-compute/stateful-actors).

## Deploy

1. **Scaffold** a new function in a scratch directory (not this cookbook directory).
   `new-func` writes your real `func_id` into the generated `func.toml`:

   ```bash
   cd /tmp   # or anywhere outside this repo
   telnyx-edge new-func -l ts -n connection-reuse
   ```

2. **Overwrite the scaffold's source files** with the cookbook versions. Replace
   `<COOKBOOK>` with the path to your local clone of `edge-compute-cookbook`:

   ```bash
   cp <COOKBOOK>/connection-reuse/index.ts \
      <COOKBOOK>/connection-reuse/package.json \
      <COOKBOOK>/connection-reuse/tsconfig.json \
      /tmp/connection-reuse/
   cd /tmp/connection-reuse
   ```

   > This replaces the scaffold's placeholder `index.ts`, `package.json`, and
   > `tsconfig.json` — expected. **Do not copy `func.toml`** — the scaffold's has your
   > real `func_id`; the cookbook's has `func_id = "REPLACE_ME"` and would break the
   > deploy.

3. **Install dependencies** (pure `node:http`, no bindings to declare):

   ```bash
   npm install
   ```

4. **Ship the function:**

   ```bash
   telnyx-edge ship    # → https://connection-reuse-<hash>.telnyxcompute.com
   ```

   > TypeScript builds typically take 10–15 minutes.

`npm run typecheck` passes as-is.

## Try it

The example caches responses from `https://httpbin.org/uuid` (a free public endpoint
that returns a fresh UUID per call). Same URL twice → cache hit, same UUID returned.
Different URLs → new cache entries, different UUIDs.

Copy the function URL from `telnyx-edge ship` output and export it so the
calls below work:

```bash
export URL=<paste function URL here>
```

```bash
# same URL twice → cache hit (same uuid returned, cachedKeys unchanged)
curl -sS $URL/foo
# → { "cachedKeys": N,   "body": "{\"uuid\":\"<uuid-A>\"}" }
curl -sS $URL/foo
# → { "cachedKeys": N,   "body": "{\"uuid\":\"<uuid-A>\"}" }  ← same uuid, same count

# different URL → new cache entry (cachedKeys +1, different uuid)
curl -sS $URL/bar
# → { "cachedKeys": N+1, "body": "{\"uuid\":\"<uuid-B>\"}" }
```

`cachedKeys` may not start at 0 on a fresh container — the platform can probe
non-`/health` paths during warm-up, and each distinct URL adds one Map entry. Watch
the *delta*, not the absolute number: same URL twice keeps it flat, a new URL bumps
it by one.
