---
title: KV read/write
---

Declare a `[storage.kv.<NAME>]` binding and `env.MY_KV` is a key-value namespace. Keys ≤ 256
chars from `a-z A-Z 0-9 - _ / = .` (colons rejected — use `/`), values ≤ 1 MiB, last-write-wins.
For counters or per-entity coordination use
[Stateful Actors](https://developers.telnyx.com/docs/edge-compute/stateful-actors) instead.

## Deploy

1. **Scaffold** a new function in a scratch directory (not this cookbook directory).
   `new-func` writes your real `func_id` into the generated `func.toml`:

   ```bash
   cd /tmp   # or anywhere outside this repo
   telnyx-edge new-func -l ts -n kv-read-write
   ```

2. **Overwrite the scaffold's source files** with the cookbook versions. Replace
   `<COOKBOOK>` with the path to your local clone of `edge-compute-cookbook`:

   ```bash
   cp <COOKBOOK>/kv-read-write/index.ts \
      <COOKBOOK>/kv-read-write/package.json \
      <COOKBOOK>/kv-read-write/tsconfig.json \
      /tmp/kv-read-write/
   cd /tmp/kv-read-write
   ```

   > This replaces the scaffold's placeholder `index.ts`, `package.json`, and
   > `tsconfig.json` — expected. **Do not copy `func.toml`** — the scaffold's has your
   > real `func_id`; the cookbook's has `func_id = "REPLACE_ME"` and would break the
   > deploy.

3. **Provision the KV namespace** and paste its ID into the scaffolded `func.toml`:

   ```bash
   telnyx-edge storage kv create --name my-cache
   # → copy the returned namespace ID
   ```

   Then append this block to the scaffold's `func.toml` (replace `<KV_ID>` with the
   returned ID):

   ```toml
   [storage.kv.MY_KV]
   id = "<KV_ID>"
   ```

4. **Generate typed bindings** — required so `env.MY_KV` type-checks:

   ```bash
   telnyx-edge types   # writes telnyx-env.d.ts (git-ignored)
   ```

5. **Ship the function:**

   ```bash
   telnyx-edge ship    # → https://kv-read-write-<hash>.telnyxcompute.com
   ```

   > TypeScript builds typically take 10–15 minutes.

Needs `@telnyx/edge-runtime` **≥ 0.2.2** (earlier silently ignore `expirationTtl`). `rest.sh`
shows the non-TypeScript REST path.

## Verify from the CLI

Anything you write from a function is readable from your laptop with `telnyx-edge storage kv`.
Useful for debugging or seeding data:

```bash
KV_ID=<your-kv-id>

# List all keys
telnyx-edge storage kv key list $KV_ID

# Read one key (mirrors env.MY_KV.get in the runtime)
telnyx-edge storage kv key get $KV_ID "user/123"

# Write one key from your laptop
telnyx-edge storage kv key put $KV_ID "user/123" '{"name":"Alice"}'

# Delete
telnyx-edge storage kv key delete $KV_ID "user/123"
```

## `expirationTtl` is server-enforced

`env.MY_KV.put(key, value, { expirationTtl: 60 })` is a server-side TTL — KV auto-deletes the
key after 60 seconds. Your code never has to check whether it's expired: subsequent `get`s
return `null` once TTL elapses. Use for OTPs, session tokens, short-lived caches.

## Handling missing keys

`env.MY_KV.get(key)` returns `null` when the key doesn't exist (or has expired). Always
null-check:

```ts
const cached = await env.MY_KV.get('user/123', { type: 'json' });
if (cached === null) {
  // key never existed or TTL expired — fetch/compute the real value
  return new Response('not found', { status: 404 });
}
return new Response(JSON.stringify(cached), {
  headers: { 'content-type': 'application/json' },
});
```
